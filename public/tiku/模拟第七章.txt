一、单选题（共20题）
1. 微信小程序代码包总大小不能超过？
A. 1MB
B. 2MB
C. 4MB
D. 8MB
答案：B
解析： 微信小程序代码包总大小不能超过 2MB。单个分包大小也不能超过 2MB，但整个小程序所有分包合计可达 20MB。超过 2MB 的主包需要进行代码压缩、资源外迁或启用分包加载。
2. 小程序分包加载中，主包和分包的关系是？
A. 主包包含所有页面，分包只是资源文件
B. 主包包含启动页和 tabBar 页，分包包含其他页面
C. 主包和分包没有区别，可以任意配置
D. 分包必须包含 tabBar 页面
答案：B
解析： 分包加载规则：主包包含 启动页面 和 tabBar 页面（必须在主包），分包包含其他非必要页面。用户进入分包页面时才下载对应分包，减少首次启动时间。subpackages 在 app.json 中配置。
3. 在 app.json 中配置分包的字段是？
A. packages
B. subpackages
C. subPackages
D. lazyPackages
答案：C
解析： 分包配置使用 subPackages（驼峰命名），数组中每个元素是一个分包配置，包含 root（分包根目录）、pages（分包页面路径列表）、name（分包别名，预下载用）等。注意是 subPackages 不是 subpackages。
4. 分包预下载的配置项是？
A. preloadRule
B. prefetchRule
C. preloadSubpackages
D. lazyLoad
答案：A
解析： preloadRule 在 app.json 中配置，指定进入某个页面时预下载哪些分包。例如："preloadRule": { "pages/index/index": { "network": "all", "packages": ["subpackage1"] } }。network 可选 wifi（仅 WiFi 预下载）或 all。
5. 小程序使用 npm 包时，需要在哪个目录下安装依赖？
A. 项目根目录
B. miniprogram 目录
C. pages 目录
D. components 目录
答案：B
解析： 小程序使用 npm 包需要在 miniprogram 目录（或项目根目录，取决于项目结构）下执行 npm install，然后在微信开发者工具中点击"工具 → 构建 npm"。构建后会生成 miniprogram_npm 目录，小程序才能正确引用。
6. 代码注入（Code Injection）在小程序中的主要作用是？
A. 向服务器注入恶意代码
B. 将公共代码提取到主包，减少分包重复代码
C. 动态加载远程代码
D. 修改小程序框架源码
答案：B
解析： 小程序的代码注入是指通过配置 lazyCodeLoading（按需注入）或将公共组件、工具代码提取到主包，避免多个分包重复包含相同代码，从而减小程序包体积。lazyCodeLoading: "requiredComponents" 是常用的按需注入配置。
7. 小程序开启按需注入（lazyCodeLoading）的配置值是？
A. true
B. false
C. "requiredComponents"
D. "all"
答案：C
解析： app.json 中配置 "lazyCodeLoading": "requiredComponents" 表示按需注入，即用户进入页面时只注入当前页面需要的自定义组件代码，而非一次性注入所有组件。这可以显著减少启动时间和内存占用。
8. 小程序性能优化中，图片资源优化的最佳实践是？
A. 将所有图片放在本地代码包中
B. 使用云存储或 CDN 存放图片，小程序中使用网络地址
C. 使用超大尺寸图片保证清晰度
D. 图片不做任何压缩处理
答案：B
解析： 图片资源应使用 云存储或 CDN 存放，小程序中通过 https:// 地址引用。这样可以减少代码包体积，利用 CDN 加速加载，支持懒加载。本地图片只适合小图标，大图必须外迁。同时应对图片进行压缩和适配（如使用 webp 格式）。
9. 小程序启动性能优化中，initialRenderingCache 的作用是？
A. 缓存网络请求数据
B. 缓存页面初始渲染结果，加速二次打开
C. 缓存用户登录态
D. 缓存本地存储数据
答案：B
解析： initialRenderingCache（初始渲染缓存）是小程序框架提供的性能优化能力，会缓存页面的初始渲染结果（WXML 首次渲染的虚拟 DOM），下次打开同一页面时直接使用缓存，跳过首次渲染过程，显著提升二次打开速度。
10. 小程序中使用 npm 包后，需要在开发者工具中执行的操作是？
A. 直接运行 npm start
B. 点击"构建 npm"
C. 手动复制 node_modules 到项目目录
D. 修改 package.json 中的 main 字段
答案：B
解析： 在小程序项目中安装 npm 包后，必须在微信开发者工具中点击 "工具 → 构建 npm"，工具会将 npm 包转换为小程序可识别的格式，生成 miniprogram_npm 目录。直接引用 node_modules 中的包小程序无法识别。
11. 分包加载中，用户首次进入分包页面时会发生什么？
A. 直接显示页面，无需下载
B. 先下载分包代码，再渲染页面
C. 自动跳转到主包页面
D. 提示用户手动下载
答案：B
解析： 分包页面在用户首次访问时，会先 异步下载分包代码，下载完成后才渲染页面。因此分包页面首次打开可能有短暂延迟。可以通过 分包预下载（preloadRule）在进入主包某些页面时提前下载分包，优化体验。
12. 小程序代码压缩优化中，JS 代码通常使用？
A. UglifyJS 或 Terser
B. 手动删除注释
C. 不需要压缩
D. 仅删除空格
答案：A
解析： 小程序构建时通常使用 UglifyJS 或 Terser 等工具进行 JS 代码压缩（删除注释、空格、缩短变量名、删除死代码等）。微信开发者工具在"详情 → 本地设置"中勾选"上传代码时自动压缩混淆"即可自动处理。
13. 小程序性能优化中，减少 setData 调用次数的方法是？
A. 每次只更新一个字段，多次调用
B. 合并数据更新，一次 setData 设置多个字段
C. 不使用 setData，直接修改 data
D. 使用定时器频繁调用 setData
答案：B
解析： setData 是小程序 JS 层到渲染层的通信接口，频繁调用会造成性能损耗。最佳实践是 合并数据更新，将多个字段的修改合并为一次 setData 调用。例如：this.setData({ a: 1, b: 2, c: 3 }) 优于三次单独调用。
14. 小程序中，WXSS 样式优化的措施包括？
A. 使用大量全局样式
B. 删除未使用的样式，压缩样式文件
C. 每个组件都写重复样式
D. 使用复杂的选择器嵌套
答案：B
解析： WXSS 优化措施：删除未使用的样式代码、压缩样式文件（去除空格和注释）、提取公共样式到 app.wxss、避免过于复杂的选择器嵌套（影响渲染性能）、使用 rpx 替代 px 减少适配代码。
15. 小程序全栈开发中，云函数的优势不包括？
A. 无需搭建服务器
B. 自动弹性伸缩
C. 前端代码完全保密
D. 免费额度内零成本起步
答案：C
解析： 云函数优势：无需购买服务器（Serverless）、自动弹性伸缩、有免费调用额度。C 错误：前端代码无法完全保密（可被反编译），敏感逻辑应放在云函数中。云函数可以保护关键业务逻辑和数据库操作。
16. 小程序性能优化的"骨架屏"技术主要用于？
A. 减少网络请求
B. 提升首屏加载体验，减少白屏时间
C. 压缩图片资源
D. 优化数据库查询
答案：B
解析： 骨架屏（Skeleton Screen）是在页面数据加载完成前显示的占位布局（灰色块/动画），让用户感知到页面正在加载，减少白屏等待的焦虑感。属于 首屏体验优化 手段，不影响实际加载速度但提升感知速度。
17. 小程序中，使用 recycle-view 长列表组件的主要目的是？
A. 美化列表样式
B. 只渲染可视区域节点，优化长列表性能
C. 增加列表动画效果
D. 实现列表拖拽排序
答案：B
解析： recycle-view（回收列表）是小程序官方提供的长列表性能优化组件，核心原理是 只渲染可视区域和缓冲区内的节点，滚动时回收离开可视区域的节点复用，避免一次性渲染大量节点导致内存溢出和卡顿。适合数据量大的列表场景。
18. 小程序构建 npm 时，哪些包可能无法直接使用？
A. 所有 npm 包都可以直接使用
B. 依赖浏览器 DOM 或 Node.js 核心模块的包
C. 纯数学计算库
D. 日期处理库
答案：B
解析： 小程序运行环境不是浏览器也不是完整 Node.js，因此依赖浏览器 DOM API（如 document、window）或 Node.js 核心模块（如 fs、path、http）的 npm 包无法直接使用。纯工具库（如 lodash、dayjs、mathjs）通常可以正常使用。
19. 小程序性能监控中，可用于分析页面渲染性能的 API 是？
A. wx.getPerformance()
B. wx.getSystemInfo()
C. wx.getBatteryInfo()
D. wx.getNetworkType()
答案：A
解析： wx.getPerformance() 获取小程序性能数据，返回包括启动耗时、页面切换耗时、渲染耗时等指标，用于性能监控和优化分析。getSystemInfo 获取设备信息，getBatteryInfo 获取电量，getNetworkType 获取网络类型，都不是性能分析 API。
20. 小程序全栈开发中，云开发数据库索引的作用是？
A. 美化数据展示
B. 提升查询性能，减少查询耗时
C. 增加数据存储容量
D. 限制数据写入速度
答案：B
解析： 云开发数据库索引用于 提升查询性能。对经常用于查询条件的字段创建索引，可以显著减少查询耗时（尤其是大数据量时）。但索引会增加写入时的开销（需要维护索引结构），因此只对高频查询字段创建索引。
二、多选题（共10题，每题4个选项）
1. 小程序分包加载的优势包括？（多选）
A. 减少主包体积，加快启动速度
B. 按需加载页面，降低首次下载时间
C. 分包合计可达 20MB
D. 所有页面都可以放在分包中
答案：A、B、C
解析： 分包优势：① 主包控制在 2MB 内，加快启动；② 页面按需下载；③ 所有分包合计可达 20MB。D 错误：tabBar 页面和启动页面必须在主包中，不能放在分包。
2. 小程序性能优化的方向包括？（多选）
A. 启动性能优化
B. 运行时性能优化
C. 网络请求优化
D. 渲染性能优化
答案：A、B、C、D
解析： 小程序性能优化涵盖：① 启动性能（代码包体积、分包、预下载、骨架屏）；② 运行时性能（setData 优化、事件节流、内存管理）；③ 网络请求（合并请求、缓存策略、CDN）；④ 渲染性能（长列表优化、减少节点数量、避免复杂动画）。
3. 代码注入（Code Injection）相关的配置包括？（多选）
A. lazyCodeLoading: "requiredComponents"
B. initialRenderingCache
C. 组件按需注入
D. 页面预渲染
答案：A、C
解析： 代码注入优化：lazyCodeLoading: "requiredComponents"（按需注入组件代码）、组件按需注入（非必要组件延迟加载）。initialRenderingCache 是渲染缓存，pagePreload 是页面预渲染，属于其他优化手段。
4. 小程序使用 npm 包的注意事项包括？（多选）
A. 在 miniprogram 目录下安装
B. 开发者工具中构建 npm
C. 包不能依赖浏览器 DOM
D. 所有 npm 包都可以直接使用
答案：A、B、C
解析： 使用 npm 包注意：① 在正确目录安装；② 必须构建 npm；③ 包不能依赖浏览器 DOM 或 Node.js 核心模块。D 错误：不是所有包都能直接用，需要确认兼容性。
5. 小程序网络请求优化方案包括？（多选）
A. 合并请求，减少请求次数
B. 使用缓存，避免重复请求
C. 图片等资源使用 CDN
D. 所有请求都使用 POST
答案：A、B、C
解析： 网络优化：合并请求、数据缓存、CDN 加速、请求去重、懒加载、分页加载。D 错误：应根据 RESTful 规范选择合适方法（GET 获取、POST 提交），不是都用 POST。
6. 小程序启动性能优化手段包括？（多选）
A. 启用分包加载
B. 代码压缩混淆
C. 资源文件外迁到云存储
D. 使用骨架屏
答案：A、B、C、D
解析： 四种都是启动优化手段：分包减少主包体积、代码压缩减少文件大小、资源外迁减少代码包、骨架屏提升首屏感知体验。此外还有预下载、按需注入、初始渲染缓存等。
7. 小程序 setData 性能优化措施包括？（多选）
A. 合并多次数据更新为一次 setData
B. 只更新变化的数据字段
C. 避免在滚动事件中频繁 setData
D. 数据量过大时进行分页或切片处理
答案：A、B、C、D
解析： setData 优化：① 合并更新减少通信次数；② 精确更新避免全量替换；③ 高频场景（如滚动）使用节流；④ 大数据量分页/切片，避免一次传输过多数据。setData 单次数据量建议不超过 256KB。
8. 小程序长列表性能优化方案包括？（多选）
A. 使用 recycle-view 回收列表
B. 分页加载数据
C. 图片懒加载
D. 一次性渲染所有数据
答案：A、B、C
解析： 长列表优化：① recycle-view 只渲染可视区域；② 分页/无限滚动避免一次加载过多；③ 图片懒加载（lazy-load 属性）。D 错误：一次性渲染所有数据会导致内存溢出和严重卡顿。
9. 小程序 WXSS 样式优化措施包括？（多选）
A. 提取公共样式到 app.wxss
B. 使用 rpx 替代 px
C. 删除未使用的样式代码
D. 避免过深层级的选择器嵌套
答案：A、B、C、D
解析： WXSS 优化：公共样式复用、rpx 自适应、删除死代码、简化选择器层级。此外还可以压缩样式文件、使用样式隔离避免全局污染。
10. 小程序全栈开发中，云开发的优势包括？（多选）
A. 前后端一体化，降低开发门槛
B. 自动弹性伸缩，无需运维
C. 免费额度内零成本起步
D. 完全替代传统服务器架构
答案：A、B、C
解析： 云开发优势：一体化开发、Serverless 免运维、免费额度。D 错误：云开发适合中小型应用，对于超大规模、复杂业务、特殊合规要求的场景，传统服务器架构仍有优势，两者可以结合使用。
