一、单选题（共30题）
1. 微信小游戏中创建 Canvas 上下文使用的 API 是？
A. wx.createCanvasContext()
B. wx.createCanvas()
C. wx.getCanvasContext()
D. wx.newCanvas()
答案：A
解析： 微信小程序/小游戏中使用 wx.createCanvasContext('canvasId') 创建画布上下文，返回 CanvasContext 对象，可进行绘图操作。基础库 2.9.0 后推荐使用 wx.createSelectorQuery() 获取 Canvas 节点，使用 2D 或 WebGL 类型上下文。
2. Canvas 中绘制矩形填充的方法是？
A. ctx.fillRect(x, y, width, height)
B. ctx.drawRect(x, y, width, height)
C. ctx.rect(x, y, width, height)
D. ctx.strokeRect(x, y, width, height)
答案：A
解析： ctx.fillRect(x, y, width, height) 绘制填充矩形。ctx.rect() 只创建路径不绘制，需配合 fill() 或 stroke()；ctx.strokeRect() 绘制空心矩形（边框）。drawRect() 不是 Canvas API。
3. 小游戏中实现游戏循环（逐帧更新）通常使用的方法是？
A. requestAnimationFrame()
B. setInterval()
C. setTimeout()
D. wx.onFrame()
答案：A
解析： requestAnimationFrame(callback) 是浏览器/小游戏标准的动画循环 API，与屏幕刷新率同步（通常 60fps），性能优于 setInterval/setTimeout。小游戏中没有 wx.onFrame() API。
4. 在 Canvas 中绘制图片使用的方法是？
A. ctx.drawImage(image, x, y, width, height)
B. ctx.putImage(image, x, y)
C. ctx.loadImage(image, x, y)
D. ctx.showImage(image, x, y)
答案：A
解析： ctx.drawImage(image, dx, dy, dWidth, dHeight) 在 Canvas 上绘制图片。支持 3 参数（仅位置）、5 参数（位置+尺寸）、9 参数（裁剪+绘制）三种重载。putImageData() 是绘制像素数据，不是图片对象。
5. 小游戏中预加载图片资源通常使用？
A. wx.loadImage() 或 new Image()
B. wx.downloadFile()
C. wx.getImageInfo()
D. wx.preloadImage()
答案：A
解析： 小游戏中使用 new Image() 创建图片对象并设置 src 属性预加载，或使用 wx.loadImage()（小游戏专用）。图片加载完成后在 onload 回调中使用。wx.downloadFile() 下载文件到临时路径，wx.getImageInfo() 获取图片信息，wx.preloadImage() 不存在。
6. 游戏精灵（Sprite）的核心属性通常不包括？
A. x、y 坐标
B. width、height 尺寸
C. src 图片路径
D. database 数据库连接
答案：D
解析： 游戏精灵（Sprite）是游戏中的基本元素，核心属性包括：位置（x, y）、尺寸（width, height）、图片资源（src/image）、速度（speedX, speedY）、可见性（visible）等。database 数据库连接与精灵概念无关。
7. Canvas 中清除画布内容的方法是？
A. ctx.clearRect(x, y, width, height)
B. ctx.clean()
C. ctx.remove()
D. ctx.erase()
答案：A
解析： ctx.clearRect(x, y, width, height) 清除指定矩形区域内的内容。通常使用 ctx.clearRect(0, 0, canvas.width, canvas.height) 清除整个画布。clean()、remove()、erase() 不是 Canvas 2D API。
8. 小游戏中创建背景音乐音频上下文使用？
A. wx.createInnerAudioContext()
B. wx.createBackgroundAudioManager()
C. new Audio()
D. wx.playMusic()
答案：A
解析： 小游戏中使用 wx.createInnerAudioContext() 创建音频上下文，可同时创建多个实例分别控制背景音乐和音效。wx.createBackgroundAudioManager() 是小程序后台音频（退出小程序仍播放），new Audio() 是浏览器 API 小游戏不支持，wx.playMusic() 不存在。
9. 玩家飞机上下移动通常通过监听什么事件实现？
A. 触摸/点击事件（touchstart/touchmove/touchend）
B. 键盘事件（keydown/keyup）
C. 陀螺仪事件（deviceorientation）
D. 摇杆事件（joystick）
答案：A
解析： 手机端小游戏通过触摸事件控制：touchstart（按下）、touchmove（拖动）、touchend（抬起）。监听触摸位置变化计算飞机移动距离。键盘事件在手机上不适用，陀螺仪用于倾斜控制，摇杆需要自定义实现。
10. 子弹从玩家飞机发射，其初始位置应该设置为？
A. 玩家飞机的中心点或顶部
B. 屏幕左上角 (0, 0)
C. 屏幕底部中央
D. 随机位置
答案：A
解析： 子弹应从玩家飞机的发射点（通常是飞机顶部中心或机翼位置）射出，即 bullet.x = player.x + player.width/2 - bullet.width/2; bullet.y = player.y。这样视觉上子弹是从飞机发射的，符合玩家预期。
11. 游戏中检测子弹与敌机碰撞通常使用？
A. 矩形碰撞检测（AABB）
B. 像素级碰撞检测
C. 圆形碰撞检测
D. 以上都可以
答案：A
解析： 2D 射击游戏中最常用的是轴对齐边界框（AABB）矩形碰撞检测，判断两个矩形是否重叠：rect1.x < rect2.x + rect2.width && rect1.x + rect1.width > rect2.x && rect1.y < rect2.y + rect2.height && rect1.y + rect1.height > rect2.y。简单高效，适合大多数情况。
12. 敌机从屏幕上方随机位置出现，其初始 x 坐标应该？
A. Math.random() * canvasWidth
B. Math.random() * enemyWidth
C. 固定为屏幕中央
D. 固定为 0
答案：A
解析： 敌机初始 x 坐标应在屏幕宽度范围内随机生成：Math.random() * (canvasWidth - enemyWidth)，确保敌机完全在屏幕内。y 坐标通常为负值（屏幕上方外），使其逐渐进入屏幕。
13. Canvas 中保存当前绘图状态的方法是？
A. ctx.save()
B. ctx.push()
C. ctx.store()
D. ctx.backup()
答案：A
解析： ctx.save() 保存当前绘图状态（变换矩阵、裁剪区域、样式属性等）到栈中，ctx.restore() 恢复最近一次保存的状态。常用于临时变换坐标系绘制旋转/缩放对象后恢复。push()、store()、backup() 不是 Canvas API。
14. 小游戏中实现爆炸动画通常使用？
A. 帧动画（逐帧切换图片）
B. CSS 动画
C. GIF 图片
D. SVG 动画
答案：A
解析： Canvas 游戏中使用帧动画实现爆炸效果：准备一组爆炸序列帧图片，按固定时间间隔逐帧绘制。CSS 动画和 SVG 动画不适用于 Canvas 环境，GIF 图片在 Canvas 中只能显示静态第一帧。
15. 游戏结束弹窗显示"重新开始"按钮，该按钮通常使用？
A. Canvas 绘制矩形+文字模拟按钮
B. HTML button 元素
C. 小程序 button 组件
D. 图片按钮
答案：A
解析： 小游戏通常完全基于 Canvas 渲染，按钮通过 Canvas API 绘制矩形背景+文字，并监听触摸事件判断点击区域。小游戏中不使用 HTML 元素或小程序组件，图片按钮也是可行方案但灵活性不如 Canvas 绘制。
16. 游戏中背景音乐应该设置为？
A. loop: true（循环播放）
B. loop: false（播放一次）
C. autoplay: false（不自动播放）
D. volume: 0（静音）
答案：A
解析： 背景音乐应设置 loop: true 循环播放，autoplay: true 自动播放（但受限于平台策略可能需要用户交互后播放）。volume 控制音量大小，通常不为 0。
17. 子弹移动的速度通常通过什么方式控制？
A. 每帧更新 bullet.y -= speed
B. 使用 CSS transition
C. 使用 wx.animate()
D. 使用 setInterval 改变位置
答案：A
解析： 游戏循环中每帧更新子弹位置：bullet.y -= speed（向上移动），然后重新绘制。Canvas 动画不使用 CSS transition 或 wx.animate()，setInterval 控制不精确且性能差。
18. 游戏暂停功能实现时，应该？
A. 取消 requestAnimationFrame 的循环或设置标志位跳过更新
B. 直接退出游戏
C. 隐藏 Canvas
D. 销毁所有游戏对象
答案：A
解析： 暂停游戏应设置 isPaused = true 标志位，在游戏循环中判断：若暂停则只绘制不更新逻辑（或完全停止循环）。不应退出游戏或销毁对象，恢复时从暂停状态继续。
19. Canvas 中设置填充颜色的属性是？
A. ctx.fillStyle

A. Game 类单例模式
B. 多个独立的 Page
C. 每次重新创建实例

答案：A

22. 游戏中同时存在多颗子弹，通常使用什么数据结构管理？
A. 数组（Array）
B. 对象（Object）
C. Set
D. Map
答案：A

25. 敌机被击中后播放爆炸音效，应该？

B. 使用背景音乐实例播放
C. 每次重新加载音频文件
D. 不播放音效
答案：A

26. 游戏分数通常存储在？
A. 全局变量或 Game 实例属性中
B. localStorage 中（仅游戏结束保存）
C. 数据库中（实时同步）
D. 文件中
答案：A
解析： 游戏进行中分数存储在内存变量中（如 this.score），每帧更新并绘制到 Canvas。游戏结束后再考虑持久化存储（如 wx.setStorageSync 保存最高分或上传到云数据库）。实时同步数据库影响性能且无必要。
27. Canvas 中旋转绘制对象前需要？
A. 先 translate 到旋转中心，再 rotate，然后绘制
B. 直接 rotate 然后绘制
C. 使用 CSS transform
D. 使用 ctx.rotation 属性
答案：A
解析： Canvas 旋转以原点 (0,0) 为中心，通常先 translate(x, y) 移动到对象中心，再 rotate(angle) 旋转，最后以相对坐标绘制（如 fillRect(-width/2, -height/2, width, height)）。绘制后 restore() 恢复状态。
28. 小游戏中"重新开始"功能的实现应该？
A. 重置游戏状态（分数、生命值、对象数组等）并重新启动游戏循环**
B. 重新加载整个小游戏
C. 跳转到新页面
D. 刷新 Canvas
答案：A
解析： 重新开始应重置游戏状态：score = 0, lives = 3, 清空 bullets/enemies 数组, 重置玩家位置, 重新启动游戏循环。不应重新加载整个游戏（体验差、加载慢），也不应跳转页面。
29. 游戏中实现难度递增（如敌机速度加快、出现频率增加）通常通过？
A. 根据分数或时间动态调整生成参数
B. 固定不变的参数
C. 随机变化无规律
D. 用户手动选择难度
答案：A
解析： 难度递增通常基于分数或时间：score > 100 时 enemySpeed += 1, spawnInterval -= 200ms 等。这样游戏有挑战性且节奏合理。固定参数太简单，随机无规律体验差，手动选择是额外功能不是核心机制。
30. 小游戏中绘制半透明颜色应使用？

1. 微信小游戏 Canvas 2D 上下文中常用的绘图方法包括？（多选）
A. fillRect() / strokeRect()
B. fillText() / strokeText()
C. drawImage()
D. arc() / fill()（绘制圆形）
答案：A、B、C、D
解析： Canvas 2D 常用绘图 API：矩形（fillRect/strokeRect/clearRect）、文字（fillText/strokeText）、图片（drawImage）、路径（arc/rect/lineTo + fill/stroke）等。这些是游戏开发中绘制背景、角色、UI 的基础。
2. 游戏开发中 requestAnimationFrame 的优势包括？（多选）
A. 与屏幕刷新率同步，动画更流畅
B. 页面不可见时自动暂停，节省性能
C. 比 setInterval 更精确
D. 支持回调函数传入时间戳参数
答案：A、B、C、D
解析： requestAnimationFrame 优势：① 与显示器刷新率同步（通常 60fps），避免丢帧和撕裂；② 标签页后台运行时自动节流/暂停；③ 由浏览器/引擎调度，比 setInterval 更稳定；④ 回调接收 DOMHighResTimeStamp 时间戳，便于计算帧间隔和动画进度。
3. 游戏精灵（Sprite）类通常包含的方法包括？（多选）
A. update()（更新位置/状态）
B. draw()（绘制到 Canvas）
C. destroy()（销毁/移除）
D. checkCollision()（碰撞检测）
答案：A、B、C、D
解析： 游戏精灵基类设计：update(dt) 每帧更新位置和状态；draw(ctx) 绘制自身；destroy() 标记删除或从对象池移除；checkCollision(other) 与其他精灵检测碰撞。还可能包含 init()、reset() 等生命周期方法。
4. 小游戏中预加载资源通常包括？（多选）
A. 图片资源（玩家、敌机、背景、爆炸序列帧）
B. 音频资源（背景音乐、射击音效、爆炸音效）
C. 字体资源（如需自定义字体）
D. 配置文件（关卡数据、游戏参数）
答案：A、B、C、D
解析： 游戏启动时应预加载：图片（精灵图、背景图、UI 图）、音频（BGM、音效）、字体（特殊字体需提前加载）、JSON 配置（关卡、敌机属性、掉落概率等）。可使用 Promise.all() 等待全部加载完成后再进入游戏。
5. 子弹对象通常包含的属性包括？（多选）
A. x、y（当前位置）
B. width、height（尺寸）
C. speed（移动速度）
D. active（是否激活/出界标记）
答案：A、B、C、D
解析： 子弹属性：位置（x, y）、尺寸（width, height，用于碰撞检测）、速度（speed 或 speedX/speedY）、激活状态（active，出界或碰撞后设为 false 待移除）、图片资源（image）、伤害值（damage）等。
6. 敌机对象的 AI 行为通常包括？（多选）
A. 直线下落（y 坐标增加）
B. 左右摇摆（x 坐标正弦波动）
C. 朝向玩家移动（追踪）
D. 随机变速
答案：A、B、C、D
解析： 敌机 AI 设计：① 基础型：直线下落；② 摇摆型：x += Math.sin(time) * amplitude；③ 追踪型：计算与玩家角度后朝玩家移动；④ 变速型：随机改变速度增加不可预测性。不同类型敌机组合增加游戏丰富度。
7. Canvas 中实现动画效果的技术包括？（多选）
A. 逐帧动画（切换序列帧图片）
B. 补间动画（计算中间状态）
C. 粒子动画（大量小图形运动）
D. 骨骼动画（复杂角色动画）
答案：A、B、C、D
解析： Canvas 动画技术：① 逐帧动画（如爆炸效果，切换 8-16 帧图片）；② 补间动画（计算起始到结束的中间状态）；③ 粒子系统（爆炸碎片、尾焰效果）；④ 骨骼动画（复杂角色，小游戏较少使用）。飞机大战常用前三种。
8. 小游戏中音频管理需要注意的问题包括？（多选）
A. 背景音乐和音效使用不同音频实例
B. 音效频繁播放需使用对象池或限制同时播放数
C. 注意 iOS 静音开关和自动播放策略限制
D. 音频资源应预加载避免播放延迟
答案：A、B、C、D
解析： 音频管理：① BGM 和 SFX 分离控制（音量独立、暂停不影响）；② 爆炸等音效可能同时多个，需限制并发数或使用对象池；③ iOS 需用户交互后才能播放音频，自动播放受限；④ 预加载避免首次播放卡顿。

12. 游戏性能优化的常用手段包括？（多选）
A. 对象池技术（重复利用子弹、敌机对象）
B. 离屏 Canvas 预渲染静态背景
C. 减少每帧绘制数量（只绘制屏幕内对象）
D. 使用整数坐标避免子像素渲染
答案：A、B、C、D
解析： 性能优化：① 对象池避免频繁创建销毁对象（减少 GC）；② 离屏 Canvas 缓存静态元素（如背景、不变的 UI）；③ 裁剪屏幕外对象（判断 x/y 是否在可视范围）；④ 整数坐标（Math.floor）避免抗锯齿带来的性能损耗。

15. 小游戏中触摸事件处理需要注意的问题包括？（多选）

16. 游戏中实现敌机生成逻辑通常涉及？（多选）
A. 随机时间间隔生成（避免过于规律）
B. 随机类型选择（不同敌机属性）
C. 随游戏进度增加生成频率
D. 生成位置在屏幕顶部随机 x 坐标
答案：A、B、C、D
解析： 敌机生成：① 时间间隔随机（如 1000ms ± 500ms）避免机械感；② 随机选择敌机类型（普通/快速/坦克等）；③ 难度递增（间隔缩短、速度加快）；④ 位置随机但确保在屏幕宽度内（x: 0 ~ canvasWidth - enemyWidth, y: -enemyHeight）。
17. Canvas 中实现视差滚动背景（Parallax）可以通过？（多选）
A. 多层背景以不同速度移动
B. 背景图纵向循环拼接
C. 使用两个背景图交替滚动
D. 静态背景不移动
答案：A、B、C
解析： 视差背景：① 远景慢速、近景快速，增加立体感；② 单张背景图循环：bgY1 += speed, bgY2 = bgY1 - bgHeight，当 bgY1 > canvasHeight 时重置到 -bgHeight；③ 两张图交替实现无缝滚动。静态背景缺乏动感。

20. 小游戏中处理屏幕适配的方案包括？（多选）
A. 设计基准分辨率，按比例缩放 Canvas
B. 动态计算元素位置（基于屏幕宽高百分比）
C. 使用 cover 或 contain 策略处理背景图
D. 为不同分辨率提供多套资源
答案：A、B、C、D
解析： 屏幕适配：① 以 iPhone 6/7/8 的 375×667 为基准，其他设备按比例缩放；② 元素位置使用百分比（如 x = canvasWidth * 0.5）而非绝对值；③ 背景图 cover（铺满可能裁剪）或 contain（完整显示可能有黑边）；④ 高清屏提供 2x/3x 资源（但小游戏通常一套图缩放即可）
