一、单选题（共20题）
1. 小程序自定义组件的文件组成不包括？
A. .js
B. .json
C. .wxml
D. .vue
答案：D
解析： 小程序自定义组件由四个文件组成：.js（组件逻辑）、.json（组件配置，需声明 "component": true）、.wxml（组件结构）、.wxss（组件样式）。.vue 是 Vue 框架的单文件组件格式，小程序不使用。
2. 自定义组件的 .json 配置文件中，必须声明的字段是？
A. "component": true
B. "usingComponents": {}
C. "page": true
D. "type": "component"
答案：A
解析： 自定义组件的 .json 文件中必须声明 "component": true，表示这是一个组件而非页面。usingComponents 是用于页面或组件中引入其他组件的配置，不是组件自身必须声明的。
3. 自定义组件中，用于接收父组件传入数据的属性定义在？
A. data
B. properties
C. methods
D. observers
答案：B
解析： properties 用于定义组件的属性（接收父组件传入的数据），支持类型、默认值和观察者函数。data 是组件内部私有数据，methods 定义组件方法，observers 用于监听数据字段变化。
4. 自定义组件中，properties 定义属性时支持的类型不包括？
A. String
B. Number
C. Function
D. Object
答案：C
解析： properties 支持的数据类型包括：String、Number、Boolean、Object、Array、null（表示不限制类型）。不支持 Function，组件间不能通过属性传递函数，需要通过事件通信。
5. 自定义组件中，监听数据字段变化使用的配置项是？
A. watch
B. observers
C. computed
D. listeners
答案：B
解析： observers 用于监听 data 和 properties 中字段的变化，类似 Vue 的 watch。格式为：observers: { '字段名': function(newVal) { ... } }。watch 和 computed 是 Vue 的概念，listeners 不是小程序组件的配置项。
6. 自定义组件的生命周期中，组件实例进入页面节点树时触发的是？
A. created
B. attached
C. ready
D. moved
答案：B
解析： attached 在组件实例进入页面节点树时触发，此时可以访问 this.data 和进行 DOM 操作（但节点可能未完全渲染）。created 在组件实例刚创建时触发（此时还不能访问 this.setData），ready 在组件布局完成后触发，moved 在组件被移动到另一个节点时触发。
7. 父组件向子组件传递数据，正确的方式是？
A. 通过 this.data 直接赋值
B. 通过属性绑定：<child name="{{value}}" />
C. 通过 wx.navigateTo 传参
D. 通过 localStorage 存储
答案：B
解析： 父组件通过属性绑定向子组件传值：<my-component title="{{pageTitle}}" />，子组件在 properties 中定义 title 接收。this.data 是内部数据，wx.navigateTo 是页面跳转，localStorage 是本地缓存，都不是组件间通信方式。
8. 子组件向父组件传递数据，正确的方式是？
A. 通过 properties 反向传递
B. 通过 this.triggerEvent() 触发自定义事件
C. 通过 getApp().globalData
D. 通过 wx.setStorageSync
答案：B
解析： 子组件通过 this.triggerEvent('事件名', { 数据 }) 触发自定义事件，父组件通过 bind:事件名 监听并接收数据。properties 是单向数据流（父→子），globalData 和 Storage 是全局状态方案，不是组件间直接通信方式。
9. 自定义组件中，定义组件方法的配置项是？
A. functions
B. methods
C. actions
D. handlers
答案：B
解析： methods 用于定义组件的方法，包括事件处理函数和普通方法。functions、actions、handlers 都不是小程序组件的配置项。注意：组件的 methods 中方法需要通过 this.方法名() 调用。
10. 组件生命周期中，组件布局完成后触发的是？
A. attached
B. ready
C. detached
D. created
答案：B
解析： ready 在组件布局完成后触发，此时组件的 DOM 结构已就绪，可以进行节点查询（如 this.createSelectorQuery()）。attached 是进入节点树，detached 是移除节点树，created 是实例创建。
11. 在页面中使用自定义组件，需要在哪个文件中声明？
A. 页面 .js 文件
B. 页面 .json 文件
C. 页面 .wxml 文件
D. app.json 文件
答案：B
解析： 在页面的 .json 文件中通过 usingComponents 声明使用的组件，如：{ "usingComponents": { "my-comp": "/components/my-comp/my-comp" } }。然后在 .wxml 中直接使用 <my-comp />。app.json 中也可以全局注册组件。
12. 自定义组件中，properties 的 observer 函数在什么时候触发？
A. 每次属性值变化时
B. 组件创建时
C. 组件销毁时
D. 页面切换时
答案：A
解析： properties 中定义的 observer 函数在属性值每次变化时触发，包括初始赋值和后续更新。格式：properties: { count: { type: Number, observer: function(newVal, oldVal) { ... } } }。注意：不要在 observer 中执行 setData 修改自身属性，可能导致死循环。
13. 组件间关系 relations 用于定义？
A. 组件的数据流向
B. 组件间的父子/兄弟关系
C. 组件的样式隔离
D. 组件的生命周期顺序
答案：B
解析： relations 用于定义组件间的父子关系（如 parent、child），使得父组件可以访问子组件实例，子组件可以访问父组件实例。常用于自定义表单组件、标签页组件等需要组件间协作的场景。
14. 自定义组件中，设置样式隔离的配置项是？
A. styleIsolation
B. scoped
C. cssModules
D. isolated
答案：A
解析： styleIsolation 用于控制组件样式隔离，可选值：isolated（完全隔离，默认）、apply-shared（页面样式影响组件）、shared（双向影响）。scoped 是 Vue 的概念，cssModules 是 CSS Modules 方案，isolated 是 styleIsolation 的一个值而非配置项名。
15. 父组件监听子组件自定义事件的正确写法是？
A. <child on:myevent="handleEvent" />
B. <child bind:myevent="handleEvent" />
C. <child @myevent="handleEvent" />
D. <child myevent="handleEvent" />
答案：B
解析： 小程序中使用 bind:事件名 或 bind事件名 监听自定义事件，如 <child bind:myevent="handleEvent" /> 或 <child bindmyevent="handleEvent" />。on: 和 @ 是 Vue 语法，小程序不支持。
16. 自定义组件中，获取组件节点信息使用的 API 是？
A. wx.createSelectorQuery()
B. this.createSelectorQuery()
C. wx.selectComponent()
D. this.selectComponent()
答案：B
解析： 在组件内部使用 this.createSelectorQuery() 创建选择器查询实例，然后 .in(this) 指定在组件范围内查询节点。wx.createSelectorQuery() 是在页面或全局使用。selectComponent 用于获取组件实例而非节点信息。
17. 组件生命周期 detached 在什么时候触发？
A. 组件创建时
B. 组件进入页面节点树时
C. 组件从页面节点树移除时
D. 组件数据更新时
答案：C
解析： detached 在组件实例从页面节点树移除时触发，适合进行清理工作（如取消定时器、移除事件监听、释放资源等）。与 attached 对应，一个进树一个出树。
18. 自定义组件中，外部样式类的作用是让？
A. 组件样式完全隔离
B. 父组件可以修改子组件内部样式
C. 子组件可以修改父组件样式
D. 组件样式自动继承页面样式
答案：B
解析： externalClasses 定义外部样式类，允许父组件通过传入 class 名来修改子组件内部指定节点的样式。例如子组件定义 externalClasses: ['my-class']，父组件使用 <child my-class="red-text" /> 即可影响子组件内部样式。
19. 组件的 behaviors 用于？
A. 定义组件样式
B. 代码复用，类似 mixins
C. 组件间通信
D. 定义组件模板
答案：B
解析： behaviors 是小程序的代码复用机制，类似于 Vue 的 mixins 或 React 的高阶组件。通过 Behavior() 定义可复用的属性、数据、生命周期和方法，多个组件可以引用同一个 behavior 来共享代码。
20. 自定义组件中，数据监听器 observers 监听多个字段联合变化的写法是？
A. observers: { 'a, b': function(a, b) {} }
B. observers: { 'a+b': function(val) {} }
C. observers: { ['a', 'b']: function() {} }
D. observers: { 'a && b': function() {} }
答案：A
解析： observers 支持监听多个字段，用逗号分隔：'a, b': function(newA, newB) { ... }。也支持通配符：'some.**' 监听对象内部所有字段变化。其他写法不符合小程序 observers 的语法规范。
二、多选题（共10题，每题4个选项）
1. 自定义组件的文件组成包括？（多选）
A. .js
B. .json
C. .wxml
D. .wxss
答案：A、B、C、D
解析： 小程序自定义组件由四个文件组成：.js（逻辑）、.json（配置，需声明 "component": true）、.wxml（结构）、.wxss（样式）。与页面文件组成相同，但 .json 的配置内容和逻辑文件的结构有所不同。
2. 自定义组件中 properties 支持的类型包括？（多选）
A. String
B. Number
C. Boolean
D. Object
答案：A、B、C、D
解析： properties 支持 String、Number、Boolean、Object、Array、null（表示任意类型）。不支持 Function 类型，组件间不能通过属性传递函数，需要通过事件机制通信。
3. 自定义组件的生命周期函数包括？（多选）
A. created
B. attached
C. ready
D. detached
答案：A、B、C、D
解析： 组件生命周期包括：created（实例创建，此时还不能 setData）、attached（进入页面节点树）、ready（布局完成）、moved（移动到另一节点）、detached（从节点树移除）、error（组件方法抛出错误）。四个选项都是核心生命周期。
4. 组件间通信的方式包括？（多选）
A. 父传子：properties
B. 子传父：triggerEvent
C. 父获取子实例：selectComponent
D. 全局状态：getApp().globalData
答案：A、B、C、D
解析： 四种都是组件间通信方式：① properties 属性绑定（父→子单向数据流）；② triggerEvent 触发自定义事件（子→父）；③ selectComponent 获取组件实例直接操作（父获取子）；④ getApp().globalData 或本地缓存等全局状态方案。还有 relations 定义组件关系也是一种方式。
5. 在页面中使用自定义组件的步骤包括？（多选）
A. 在页面 .json 中 usingComponents 声明
B. 在页面 .wxml 中使用组件标签
C. 在页面 .js 中 import 引入
D. 在 app.json 中全局注册
答案：A、B
解析： 使用自定义组件的标准步骤：① 在页面 .json 的 usingComponents 中声明组件路径；② 在 .wxml 中直接使用组件标签。C 错误：不需要在 .js 中 import。D 不是必须：app.json 中全局注册后所有页面可用，但也可以在页面局部注册。
6. 自定义组件中 observers 的用途包括？（多选）
A. 监听 data 字段变化
B. 监听 properties 字段变化
C. 监听多个字段联合变化
D. 监听页面生命周期
答案：A、B、C
解析： observers 用于监听 data 和 properties 中字段的变化，支持单个字段、多个字段（逗号分隔）、通配符路径（如 'some.**'）。D 错误：生命周期监听使用 lifetimes 配置项，不是 observers。
7. 组件样式隔离 styleIsolation 的可选值包括？（多选）
A. isolated
B. apply-shared
C. shared
D. global
答案：A、B、C
解析： styleIsolation 三个可选值：isolated（完全隔离，默认，页面样式不影响组件，组件样式不影响页面）、apply-shared（页面样式影响组件，组件样式不影响页面）、shared（双向影响）。没有 global 这个值。
8. 组件的 behaviors 可以包含的内容包括？（多选）
A. properties
B. data
C. methods
D. lifetimes
答案：A、B、C、D
解析： Behavior() 可以定义：properties、data、methods、behaviors（嵌套 behavior）、lifetimes（生命周期）、observers（数据监听）、relations（组件关系）等。与组件的配置项基本一致，用于代码复用。
9. 父组件获取子组件实例的方式包括？（多选）
A. this.selectComponent('#childId')
B. this.selectComponent('.child-class')
C. 通过 triggerEvent 反向获取
D. 通过 relations 定义父子关系
答案：A、B、D
解析： this.selectComponent('选择器') 通过 id 或 class 获取子组件实例，然后可以直接调用子组件方法。relations 定义父子关系后，也可以通过 this.getRelationNodes() 获取关联组件。C 错误：triggerEvent 是子向父传数据，不是获取实例。
10. 自定义组件中 externalClasses 的作用包括？（多选）
A. 允许父组件传入 class 名
B. 修改组件内部指定节点的样式
C. 完全替代组件内部样式
D. 实现组件样式的部分自定义
答案：A、B、D
解析： externalClasses 定义外部样式类，允许父组件通过属性传入 class 名，影响组件内部指定节点的样式，实现组件样式的部分自定义（而非完全替代）。C 错误：不是完全替代内部样式，而是让特定节点可以被外部定制。
