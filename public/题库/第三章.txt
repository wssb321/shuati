1. (单选题) 微信小程序全新的渲染引擎 Skyline 相比于传统的 WebView 引擎，在架构上做出了什么重大改变以提升性能?

A. 强制将渲染和业务逻辑合并在同一条线程上执行
B. 采用双线程架构，使渲染线程与逻辑线程相互独立，避免逻辑阻塞界面 ✅
C. 完全废弃了 WXML 和 WXSS 代码规范
D. 将所有的计算都转移给第三方服务器进行处理

答案：B

解析：Skyline 渲染引擎的核心改进是采用双线程架构，将渲染线程与逻辑线程分离，避免了 JavaScript 逻辑执行阻塞 UI 渲染的问题，从而大幅提升性能和流畅度。
- A 错误：Skyline 恰恰是将渲染和逻辑分离，而不是合并。
- C 错误：Skyline 仍然兼容 WXML 和 WXSS 规范。
- D 错误：计算仍然在本地进行，没有转移给第三方服务器。

2. (单选题) 在微信小程序的通信模型中，主要负责处理页面布局和样式显示的是哪一层?

A. 逻辑层
B. 渲染层 ✅
C. 物理层
D. 数据层

答案：B

解析：微信小程序采用双线程架构：
- 渲染层（View 层）：负责 WXML 和 WXSS 的解析、页面布局和样式渲染，运行在 WebView 或 Skyline 中。
- 逻辑层（App Service 层）：负责 JS 逻辑执行、数据处理，运行在 JSCore 中。
- 渲染层和逻辑层通过 Native 层进行异步通信。

3. (单选题) 当用户在使用小程序时，按下手机的 Home 键将小程序切换到后台运行，此时会触发小程序的哪一个应用级别的生命周期函数?

A. onLaunch
B. onShow
C. onReady
D. onHide ✅

答案：D

解析：应用级别的生命周期函数包括：
- onLaunch：小程序初始化完成时触发（全局只触发一次）。
- onShow：小程序启动或从后台进入前台时触发。
- onHide：小程序从前台进入后台时触发（如按 Home 键、锁屏、切换到其他应用）。✅
- onReady 是页面级别的生命周期，不是应用级别的。

4. (单选题) 在页面级别的生命周期函数中，通常建议将"页面初始化数据"(如发起网络请求获取默认数据)的操作放在哪一个函数中执行?

A. onLoad ✅
B. onShow
C. onReady
D. onUnload

答案：A

解析：页面级别的生命周期函数：
- onLoad：页面加载时触发，一个页面只会调用一次，适合进行页面初始化数据操作（如发起网络请求获取默认数据）。✅
- onShow：页面显示/切入前台时触发，每次打开页面都会触发。
- onReady：页面初次渲染完成时触发，此时可以和视图层进行交互。
- onUnload：页面卸载时触发，适合进行清理操作。

5. (单选题) 在小程序的 WXML 文件中，如果想要把定义在 JS 文件 data 中的变量 sum 动态绑定并渲染到页面上，正确的语法格式是?

A. {sum}
B. {{sum}} ✅
C. ${sum}
D. <sum>

答案：B

解析：微信小程序使用 Mustache 语法（双大括号）进行数据绑定，格式为 {{变量名}}。
- A. {sum}：单大括号不是小程序的数据绑定语法。
- B. {{sum}}：正确的 Mustache 数据绑定语法。✅
- C. ${sum}：是 ES6 模板字符串的语法，不是 WXML 的数据绑定语法。
- D. <sum>：会被当作标签名解析，不是数据绑定。

6. (单选题) 在小程序代码逻辑中，若想动态更新页面数据并实现视图层的同步渲染，必须通过以下哪种方式修改数据?

A. 直接使用 this.data.变量名 = 新值
B. 使用 this.updateData({ 变量名: 新值 })
C. 使用 this.setData({ 变量名: 新值 }) ✅
D. 修改 DOM 节点的 innerText

答案：C

解析：微信小程序中，必须使用 this.setData() 方法来更新数据并同步渲染视图层。
- A 错误：直接修改 this.data 不会触发视图更新，只是修改了数据本身。
- B 错误：不存在 this.updateData() 方法。
- C 正确：this.setData({ key: value }) 是官方提供的标准数据更新方法，会触发视图层重新渲染。✅
- D 错误：小程序没有 DOM 操作的概念，不能直接操作 DOM 节点。

7. (单选题) 如果页面中有一个元素需要非常频繁地切换显示与隐藏状态，从性能优化的角度考虑，推荐使用哪一种条件渲染方式?

A. wx:if
B. block 包裹
C. display: inline
D. hidden 属性 ✅

答案：D

解析：
- wx:if：条件为 false 时，组件会被销毁（从 DOM 树中移除）；条件变为 true 时，需要重新创建和渲染。频繁切换会导致频繁的创建和销毁，性能开销大。
- hidden：只是通过 CSS 控制 display: none 来隐藏元素，组件始终存在于 DOM 树中，只是不可见。频繁切换时性能开销小。✅
- 因此，频繁切换显示/隐藏时，推荐使用 hidden；偶尔切换或需要彻底销毁释放资源时，使用 wx:if。

8. (单选题) 使用 wx:for 进行列表渲染时，如果不使用 wx:for-item 去特别指定，默认代表循环中"当前遍历项"的变量名是?

A. index
B. item ✅
C. key
D. element

答案：B

解析：wx:for 列表渲染的默认变量名：
- 默认当前遍历项的变量名是 item。✅
- 默认当前索引的变量名是 index。
- 可以通过 wx:for-item="xxx" 自定义遍历项变量名。
- 可以通过 wx:for-index="yyy" 自定义索引变量名。

9. (单选题) 在给小程序组件绑定点击事件时，如果想要阻止该事件向父节点继续传递(即阻止事件冒泡)，应该使用以下哪种绑定语法?

A. bindtap
B. catchtap ✅
C. ontap
D. preventtap

答案：B

解析：
- bindtap：事件绑定，事件会向上冒泡传递给父节点。
- catchtap：事件捕获绑定，阻止事件向上冒泡，事件不会传递给父节点。✅
- ontap：不是小程序的事件绑定语法。
- preventtap：不是小程序的事件绑定语法（preventDefault 是 DOM 中的概念，小程序中没有 preventtap）。

10. (单选题) 在事件处理函数的 event 对象中，用来表示"当前绑定了事件处理函数的组件"的属性是哪一个?

A. target
B. type
C. timeStamp
D. currentTarget ✅

答案：D

解析：event 对象中的关键属性：
- target：触发事件的源组件（最内层实际触发事件的组件）。
- currentTarget：当前绑定了事件处理函数的组件（事件监听器所在的组件）。✅
- type：事件类型（如 tap、longpress 等）。
- timeStamp：事件触发的时间戳。

例如：父容器绑定了点击事件，子元素实际被点击时，target 指向子元素，currentTarget 指向父容器。

12. (多选题) 当使用普通的 navigateTo 方式从页面 A 跳转到页面 B 时，关于这两个页面的生命周期变化，下列描述正确的有?

A. 页面A会被销毁，触发 onUnload
B. 页面A仅是被隐藏进入页面栈，触发 onHide ✅
C. 页面B会被初始化并显示，触发 onLoad 和 onShow ✅
D. 页面A和页面B都会触发 onLaunch

答案：B、C

解析：使用 wx.navigateTo 跳转时：
- 页面 A：不会被销毁，只是被隐藏并压入页面栈，触发 onHide。✅
- 页面 B：被初始化，触发 onLoad（首次加载）和 onShow（页面显示）。✅
- A 错误：navigateTo 不会销毁原页面，redirectTo 或 navigateBack 才会导致页面卸载。
- D 错误：onLaunch 是应用级别的生命周期，只在小程序初始化时触发一次，不是页面级别的。

13. (多选题) 在使用 setData 方法更新数据时，为了保证小程序的性能并避免潜在的 Bug，应该遵循以下哪些原则?

A. 绝对不能使用 this.data 直接对变量赋值来更新视图 ✅
B. 尽量避免一次性设置过多的数据(受最大值限制) ✅
C. 避免过于频繁地调用 setData，尽量将多个赋值操作合并在一次调用中 ✅
D. 推荐将暂时不用的变量通过 setData 设置为 undefined

答案：A、B、C

解析：setData 使用最佳实践：
- A 正确：直接修改 this.data 不会触发视图更新，必须使用 this.setData()。
- B 正确：单次 setData 设置的数据量有大小限制（1MB），过多的数据会导致性能问题和传输失败。
- C 正确：频繁调用 setData 会导致频繁的通信和渲染，应该合并多次更新为一次调用。✅
- D 错误：不需要将不用的变量设为 undefined，setData 只更新需要变更的数据即可，小程序会自动处理差异更新。

14. (多选题) 关于条件渲染中的 wx:if 和 <block> 标签，下列说法正确的有?

A. <block> 标签并不是一个真正的组件，仅仅是一个包装元素 ✅
B. <block> 标签本身不会在页面上做任何的渲染输出 ✅
C. 可以将 wx:if 指令添加在 <block> 标签上，用来控制内部多块元素的渲染 ✅
D. 当 wx:if 的判断条件为 false 时，其控制的代码块元素会被直接销毁(不渲染) ✅

答案：A、B、C、D

解析：
- A 正确：<block> 是小程序提供的包装元素，不是真正的组件，不会在页面中生成实际的 DOM 节点。
- B 正确：<block> 只起到逻辑包裹作用，不做任何渲染输出。
- C 正确：可以在 <block> 上使用 wx:if、wx:for 等指令，控制内部多个元素的批量渲染。
- D 正确：wx:if 为 false 时，组件会被销毁（从视图层移除），条件变为 true 时重新创建。

15. (多选题) 在小程序的事件体系中，事件被分为了"冒泡事件"和"非冒泡事件"。下列事件中，属于"冒泡事件"(触发后会向父节点传递)的有?

A. tap (手指触摸点击) ✅
B. longpress (手指长按) ✅
C. input (输入框键盘输入)
D. submit (表单提交)

答案：A、B

解析：小程序的事件分类：
冒泡事件（触发后会向父节点传递）：
- touchstart、touchmove、touchend、touchcancel
- tap、longpress、longtap ✅
- transitionend、animationstart、animationiteration、animationend

非冒泡事件（不会向父节点传递）：
- submit（表单提交）、input（输入框输入）、change（值改变）、scroll（滚动）等。✅

因此 A（tap）和 B（longpress）属于冒泡事件，C（input）和 D（submit）属于非冒泡事件。
